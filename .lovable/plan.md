## الفكرة العامة

صفحة جديدة باسم **"تسعير العروض / صنف جديد"** تحت قسم **Recipes** في القائمة الجانبية، مسارها `/recipes/offers`.
الصفحة تتيح بناء عرض مكوّن من عدة أصناف فرعية (سندوتش 1، سندوتش 2 …) — كل صنف ليه ريسبي مستقل (إما منقول من منتج موجود في المنيو أو مكوّنات يدوية)، مع حساب تكلفة كاملة بنفس منطق **تحليل المنيو** (تعبئة + مستهلكات % + Side Cost + مصاريف غير مباشرة + ضريبة) وإخراج هامش ربح + طباعة احترافية.

---

## مكونات الصفحة

### 1) شريط أعلى الصفحة
- فلتر الفرع (نفس باقي الصفحات)
- زر **"عرض جديد"** — يفتح فورم جديد فاضي
- قائمة بالعروض المحفوظة (سايد لِست يسار) للبحث/التعديل/الحذف

### 2) رأس العرض (Header Form)
- اسم العرض، كود تلقائي `OFR_0001`
- الفرع
- سعر البيع المستهدف (يدوي)
- نوع الطلب: صالة / تيك أواي (يأثر على الضريبة)
- نسبة الضريبة (يدوي، افتراضي من الفترة)
- ملاحظات

### 3) قسم الأصناف الفرعية (Sub-items)
زر **"إضافة صنف للعرض"** يفتح Dialog فيه خيارين:
- **اختيار من منتجات المنيو** → يجيب الريسبي تلقائيًا من جدول `recipes` ويسمح بتعديل الكميات للعرض ده فقط (snapshot، مش بيأثر على الريسبي الأصلي).
- **صنف مخصص جديد** → اسم + اختيار خامات يدويًا (نفس واجهة إضافة المكونات في صفحة الريسبي).

كل صنف فرعي يظهر في **Accordion**:
- العنوان: اسم الصنف + إجمالي تكلفة الخامات
- المحتوى: جدول الخامات (الاسم، الوحدة، الكمية، م.التكلفة، الإجمالي) — قابل للتعديل/الحذف/إضافة خامة جديدة + استبدال خامة.

### 4) قسم المصاريف الغير المباشرة (يدوي)
- تكلفة التعبئة (Packing) للعرض ككل أو موزعة
- نسبة المستهلكات % (Default + إمكانية تغييرها)
- Side Cost
- نسبة المصاريف غير المباشرة % (يدخلها المستخدم — نفس قيمة "متوسط نسبة التكلفة المباشرة" من تحليل المنيو، أو يدوي)

### 5) ملخص مالي (Live Calculation)
- إجمالي تكلفة الخامات (مجموع الأصناف الفرعية)
- + التعبئة + المستهلكات + Side Cost
- = **إجمالي التكلفة المباشرة**
- + المصاريف غير المباشرة
- = **التكلفة الكاملة**
- سعر البيع، الضريبة، **Net Take Away / Net Table**
- **صافي الربح** و**هامش الربح %** بألوان (أحمر <20%، أخضر 20–40%، أصفر >40%) — حسب الذاكرة الحالية للنظام.

### 6) أزرار الحفظ والطباعة
- **حفظ** كمسودة / **حفظ نهائي**
- **طباعة احترافية**: ورقة فيها الهيدر (لوجو + اسم الشركة + اسم العرض)، جدول لكل صنف فرعي (عنوان واضح بالكاتجوري) ومكوناته وتكلفته، وفي النهاية ملخص التكلفة والربح. نفس استايل `RecipePrintExport`.
- تصدير Excel + PDF.

## التغييرات التقنية

### قاعدة البيانات (Migration)
ثلاث جداول جديدة مع RLS بنفس النمط (`company_id = get_user_company_id()`):

```text
menu_offers
 ├─ id, company_id, branch_id
 ├─ code (OFR_NNNN), name, status (مسودة/جاهز)
 ├─ sale_price, tax_rate, order_type
 ├─ packing_cost, consumables_pct, side_cost
 ├─ indirect_expenses_pct, notes
 └─ created_at, updated_at

menu_offer_items
 ├─ id, offer_id (FK), sort_order
 ├─ name
 └─ source_pos_item_id (nullable) – لو منقول من منتج موجود

menu_offer_ingredients
 ├─ id, offer_item_id (FK)
 ├─ stock_item_id, name, unit
 ├─ qty, conversion_factor
 └─ avg_cost (snapshot لتثبيت التكلفة)
```

- دالة `generate_offer_code(p_company_id)` تشبه `generate_supplier_code` لإصدار `OFR_0001`.
- RLS: 4 policies (select/insert/update/delete) لكل جدول بنفس باترن باقي الجداول.

### كود الواجهة
- ملف جديد: `src/pages/MenuOffersPage.tsx` (الصفحة الرئيسية)

- ملف جديد: `src/components/MenuOfferPrint.tsx` (HTML الطباعة بنفس استايل `RecipePrintExport`)
- إضافة المسار في `src/App.tsx`: `/recipes/offers`
- إضافة العنصر في `src/components/SystemLayout.tsx` تحت children الخاصة بـ recipes:
  ```text
  { id: "recipes-offers", path: "/recipes/offers", label: "تسعير العروض", icon: Tags }
  ```
- استخدام `useBranchCosts` لحساب التكلفة حسب الفرع (مش global) — متوافق مع باقي النظام.
- الالتزام بـ `??` بدل `||` (Core rule).

1. هيدر (لوجو + اسم العرض + التاريخ + الفرع)

3. جدول ملخص العرض: إجمالي التكلفة المباشرة، التعبئة، المستهلكات، Side Cost، المصاريف غير المباشرة، التكلفة الكاملة، سعر البيع، الضريبة، صافي الربح، هامش الربح %.

## ملاحظات مهمة
- snapshot لتكلفة الخامة وقت الحفظ (avg_cost) عشان لو تكلفة الخامات اتغيرت بعدين، التكلفة المحفوظة للعرض تفضل ثابتة. مع زر **"تحديث الأسعار من المخزون"** يدوي لو حب يعيد الحساب.
- لو المستخدم اختار منتج موجود من المنيو، بنعمل deep copy للمكونات في الـ snapshot، ومش بنغيّر الريسبي الأصلي إطلاقًا.
- العروض معزولة per-company عبر RLS وفلتر الفرع نفس بقية الصفحات.

هنبدأ التنفيذ بترتيب: الـ migration الأول → الصفحة والكود → الطباعة → الربط في القائمة الجانبية.
