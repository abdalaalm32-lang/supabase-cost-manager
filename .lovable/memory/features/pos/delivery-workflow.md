- حقل `delivery_fee` في `pos_sales` لرسوم التوصيل
- حقل `driver_id` في `pos_sales` مرتبط بجدول `delivery_drivers`
- حقل `phone2` في `customers` لرقم تليفون إضافي
- جدول `delivery_drivers` (name, phone, active, company_id) مع RLS
- **الكول سنتر يحدد الفرع** عبر dropdown إجباري عند إنشاء الأوردر (بدون تحديد الطيار)
- الكول سنتر يحدد رسوم التوصيل فقط عند إنشاء الأوردر
- **الكاشير** هو من يحدد الطيار عبر نافذة إدارة الدليفري في شاشة البيع (POS)
- شاشة البيع (POS) تعرض badge تنبيه بعدد أوردرات الدليفري النشطة مع صوت تنبيه عبر Realtime
- **الأوردرات تُفلتر حسب الفرع المحدد في POS** - كل كاشير يرى فقط أوردرات فرعه
- الضغط على التنبيه يفتح نافذة إدارة أوردرات الدليفري (تعيين طيار، تغيير الحالة، عرض الأصناف، طباعة المطبخ/الطيار)
- حالات الأوردر: جديد → قيد التحضير → خرج للتوصيل → تم التسليم
- صفحة "تسوية الطيارين" في إدارة المبيعات (/sales/driver-settlement) لعرض حسابات كل طيار يومياً
- **الطباعة الحرارية**: جميع الإيصالات تستخدم inline styles بعرض 72mm (مع @page size 80mm) للتوافق مع طابعات 80mm الحرارية
- مكون PosReceiptPrint يدعم deliveryFee, customerPhone, customerPhone2, customerAddress
